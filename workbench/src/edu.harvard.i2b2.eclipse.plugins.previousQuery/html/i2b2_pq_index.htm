Introduction

Describes the layout and contents of this view.

 

Layout of the Previous Query View

A previous query () is created automatically when a query is run from the Query Tool.  The query is stored and displayed in the Previous Query View. These previous queries can be used to run new queries or to easily populate other views such as the timeline view.

 

Figure 6. 1     Previous Query View

 

 

The query in the previous query is comprised of several levels that can be seen by expanding each level of the tree.  These levels are outlined below:

 

Previous Query Name

The name of the previous query is the top level of the hierarchal tree and is comprised of the following components.

1.       Description – abbreviated name to describe the previous query.

2.       Time – the time the query was run.

3.       Date – the date the query was run.

4.       User Id – the user who ran the query.

Note: unless you are a manager you will only see your queries.

 

Results Folder

The results folder contains the results of the query. Expand the view by clicking on the plus sign () to see the query results.

 

Results

There are two types of results; (1) Patient List and (2) Patient Count. The Timeline and Patient Count check boxes located in the Query Tool View will determine which results are included in the previous query.

 

Patient List

The Patient List () appears if Timeline was checked at the time the query was run.  The list contains all the patients that met the search criteria. Included in the list are the Patient ID and basic demographic information.

 

Figure 6. 2     Patient List

 

 

Number of Patients

The Number of Patients () appears if Patient Count was checked at the time the query was run.  The actual number is displayed in the tool tip, which can be seen by hovering over it.  The number that displays is the number of patients who met the criteria defined in the creation of the query.

 

Figure 6. 3     Number of Patients

 

 

 

Navigation

Rename a Previous Query

Users can rename a previous query from the list by selecting Rename from the drop-down list.

 

1.       Using the right mouse button, click on the Previous Query to be renamed.

2.       A drop down list will open.

 

Figure 6. 4     Rename

 

 

3.       Select Rename from the list.

 

 

Delete/Remove a Previous Query

Users can remove a previous query from the list by selecting Delete from the drop-down list.

 

1.       Using the right mouse button, click on the Previous Query to be deleted.

2.       A drop down list will open.

 

Figure 6. 5     Delete

 

 

3.       Select Delete from the list.

 

Warning: Once you click on Delete, the previous query will be removed. Therefore you need to make sure you want to remove it from the list.

 

Refresh the List

Users can refresh the list of previous queries by selecting Refresh All from the drop-down list.

 

1.       Using the right mouse button, click on any of the previous queries in the list.

2.       A drop down list will open.

 

Figure 6. 6     Refresh All

 

 

3.       Select Refresh All from the list.

 

 

Display Options

Users can define how many previous queries to display and the order in which they will appear in the list.  The display options are set in the Previous Queries Options Dialog window (Figure 6.7), which can be accessed by clicking on the Options () button.

 

Figure 6. 7     Options Dialog Box

 

 

Maximum to Display

Queries to Display

The Maximum number of queries to be displayed defines how many previous queries will display in the Previous Query View. Once the maximum number is reached the oldest query will no longer appear in the workplace.

 

Recommendation: Use the Workplace View to save those previous queries that will be used often. This will prevent the lost of the query in the event that the maximum number to display has been reached.

 

Patients to Display

The Maximum number of patients to be displayed is used when the system is retrieving the list of patients to display in the Previous Query View.  Once the maximum number is reached a message (Figure 6.8) will appear informing you that you have reached the maximum defined and that continuing to retrieve all the patients in the list may affect performance. You will have the option to continue or to cancel the action.

 

Figure 6. 8     Patient Count Message

 

 

Order of Display

Sort by Time

Selecting sort by time when the query was created will sort the queries in the Previous Query View by the date and time the query was created.  Once you select to sort by time, you need to define whether or not it will be in Ascending or Descending order.

Ascending: displays the oldest query first.

Descending: displays the newest query first.

 

Note: Renaming a query will not affect the sort order as the date and time is stored with the previous query.

 

Sort by Query Name

Selecting sort by query name will sort the queries in the Previous Query View alphabetically by the name of the query.  Once you select to sort by name, you need to define whether or not it will be in Ascending or Descending order.

Ascending: queries are displayed in alphabetical order with the beginning of the alphabet at the end of the list.  (Z to A)

Descending: queries are displayed in alphabetical order with the beginning of the alphabet at the beginning of the list. (A to Z)

 

 

 

Using a Previous Query

This section describes how to use a previous query.

 

Draggable Items

Different parts of the previous query can be dragged and dropped (copied) from the Previous Query View to other views in the i2b2 Workbench.

 

1.       Previous Query

Associated with each previous query () is the concepts used to run the query and the results. A previous query can be added to other views by dragging the previous query name from Previous Query to one of the following views.

 

a.)    Export Data View; Select Concepts Tab

§         Drop in the concept list section.

 

b.)    Query Tool View

§         Drop in the Query Name field.

 

c.)     Timeline View; Create model for Timeline Tab

§         Drop in the concept list section.

 

d.)    Workplace View

§         Drop in the user’s folder or the shared folder.

 

2.       Patient Set

A specific set of patients can be added to other views by dragging the patient list () from Previous Query to one of the following views.

 

a.)    Export Data View; Select Patients Tab

§         Drop in the patient list section.

 

b.)    Timeline View; Create model for Timeline Tab

§         Drop in the Patient Set field.

 

c.)     Workplace View

§         Drop in the user’s folder or the shared folder

 

3.       Individual Patient

An individual patient can be added to other views by dragging the patient () from Previous Query to one of the following views.

 

a.)    Export Data View; Select Patients Tab

§         Drop in the Patient List section.

 

b.)    Workplace View

§         Drop in the user’s folder or the shared folder.

 

4.       Patient Count

The patient count can be added to the Workplace View by dragging number of patients () from Previous Query view.

 

 

Add to Query Tool View

A previous query can be added to the query tool view by simply dragging the query name from Previous Query View.

 

Add Previous Query To Query Tool View

1.       Click on the previous query and drag the item over to the Query Tool View.

2.       Drop the previous query into the Query Name (Figure 6.9).

 

Figure 6. 9     Dropping Previous Query in Query Tool View

 

 

3.       The concept(s) associated with the previous query will display in the appropriate group(s) and the name of the query will appear at Query Name (Figure 6.10).

 

Figure 6. 10  Previous Query Added to Query Tool View

 

 

Add to Workplace View

Information in the workplace is related to the most common concepts and queries that an individual uses and in essence becomes their personal workplace.

 

Figure 6. 11  Workplace View

 

 

Add Previous Query To Workplace View

1.       Highlight the previous query by clicking on the name of the query.

2.       While holding the left mouse button down, drag the previous query name to the Workplace View.

3.       Drop the previous query into either your workplace folder or the shared folder.

4.       The previous query will now appear in the workplace with the same name.

 

Add Patient Set To Workplace View

1.       Click on the plus sign () next to the previous query that contains the patient set.

2.       Click on the plus sign () next to the results folder.

3.       Click on patient list ().

4.       While holding the left mouse button down, drag the patient list to the Workplace View.

5.       Drop the patient list into either your workplace folder or the shared folder.

6.       The patient list will now appear in the workplace.

 

Add A Patient To Workplace View

1.       Click on the plus sign () next to the previous query that contains the patient set.

2.       Click on the plus sign () next to the results folder.

3.       Click on the plus sign () next to patient list.

4.       Click on the patient () to be copied.

5.       While holding the left mouse button down, drag the patient to the Workplace View.

6.       Drop the patient list into either your workplace folder or the shared folder.

7.       The patient will now appear in the workplace.

 

Add Patient Count To Workplace View

1.       Click on the plus sign () next to the previous query that contains the patient set.

2.       Click on the plus sign () next to the results folder.

3.       Click on number of patients ().

4.       While holding the left mouse button down, drag the number of patients to the Workplace View.

5.       Drop the number of patients into either your workplace folder or the shared folder.

6.       The number of patients will now appear in the workplace as patient count ().

 

 

Add to Timeline View

The timeline view displays the information in a graphical representation of a chronological sequence of events.

 

Figure 6. 12  Timeline View

 

 

Add Previous Query To Timeline View

1.       Highlight the previous query by clicking on the name of the query.

2.       While holding the left mouse button down, drag the previous query name to the Timeline View.

3.       Drop the previous query into concept list section of the Create model for timeline tab.

4.       The concepts and patient set associated with the previous query will now appear in the Timeline view.

 

Add Patient Set To Timeline View

1.       Click on the plus sign () next to the previous query that contains the patient set.

2.       Click on the plus sign () next to the results folder.

3.       Click on patient list ().

4.       While holding the left mouse button down, drag the patient list to the Timeline View.

5.       Drop the patient list into the Patient Set field located on the Create model for timeline tab.

6.       The patient set field will now display the number of patients in the patient list.

 

 

Add to Export Data View

The export data view is designed to compile the requested data and display the information in a graphical representation.

 

Figure 6. 13  Export Data View

 

 

Add Previous Query To Export Data View

1.       Highlight the previous query by clicking on the name of the query.

2.       While holding the left mouse button down, drag the previous query name to the Export Data View.

3.       Drop the previous query into concept list section of the Select Concepts tab.

4.       The concepts associated with the previous query will now appear in the Export Data View.

 

Add Patient List To Export Data View

1.       Click on the plus sign () next to the previous query that contains the patient set.

2.       Click on the plus sign () next to the results folder.

3.       Click on patient list ().

4.       While holding the left mouse button down, drag the patient list to the Export Data View.

5.       Drop the patient list into the patient list section of the Select Patients tab.

6.       The patient list section will now display all the patients in the patient list.

 

Add A Patient To Export Data View

1.       Click on the plus sign () next to the previous query that contains the patient set.

2.       Click on the plus sign () next to the results folder.

3.       Click on the plus sign () next to patient list.

4.       Click on the patient () to be copied.

5.       While holding the left mouse button down, drag the patient to the Export Data View.

6.       Drop the patient into the patient list section of the Select Patients tab.

7.       The patient will now appear in the list of patients.
