Provides a java interface to the CLUTO clustering toolkit library.

CLUTO is a software package for clustering low- and high-dimensional datasets and for analyzing the characteristics of the various clusters. CLUTO is well-suited for clustering data sets arising in many diverse application areas including information retrieval, customer purchasing transactions, web, GIS, science, and biology. 

CLUTO provides three different classes of clustering algorithms that operate either directly in the object's feature space or in the object's similarity space. These algorithms are based on the partitional,  agglomerative, and graph-partitioning paradigms. A key feature in most of CLUTO's clustering algorithms is that they treat the clustering problem as an optimization process which seeks to maximize or minimize a particular  clustering criterion function defined either globally or locally over the entire clustering solution space. CLUTO provides a total of seven different criterion functions that can be used to drive both partitional and agglomerative clustering algorithms, that are described  and analyzed in [ZK01, ZK02b]. Most of these criterion functions have been shown to produce high quality clustering solutions in high dimensional datasets, especially those arising in document clustering. In addition, CLUTO provides some of the more traditional local criteria (e.g., single-link, complete-link, and UPGMA) that can be used in the context of agglomerative clustering. Furthermore, CLUTO provides graph-partitioning-based clustering algorithms that are well-suited for finding clusters that form contiguous regions that span different dimensions of the underlying feature space

JCluto

The jcluto package is written to use java interfaces as method parameters and return values so that alternate implementations of specific classes are accommodated.

Access to the CLUTO library functions is through methods in JClutoWrapper.java. The Java Native Interface code in JClutoWrapper.c maps the java methods to C functions.

The graph-based cluto functions have not been tested yet, and the jcluto package does not yet offer parameter settings to call those functions.

The CLUTO library was intended to be used in command line applications, and it will perform a system exit whenever it encounters parameters that it cannot handle.

JCluto encpsulates the matrix format (rowptr, rowind, rowval in Section 5.2 of manual.pdf) in the ClutoMatrix interface. The ClutoMatrix is implemented in the ClutoTableMatrix such that it also supports the javax.swing.table.TableModel interface, making it easy to display a cluto matrix in a javax.swing.JTable.

The TransMatrixPanel provides a GUI interface to routines that can transform the ClutoMatrix. These include missing value estimation to generate a dense matrix from a sparse one. It can also perform a log transform on the matrix values.

The ClutoOptionsPanel provides a GUI interface for setting the paramters for invoking a CLUTO clustering solution. This interface attempts to set parameters that are consistent with the chosen clustering method.

The ClutoCluster class executes the CLUTO clustering method. The method getParts() returns an int array that give the cluster/part number the each row in the matrix was assigned to in the clustering solution. The edu.umn.genomics.table.cluster.cluto package provides examples of displaying and visualizing the results of clustering.
