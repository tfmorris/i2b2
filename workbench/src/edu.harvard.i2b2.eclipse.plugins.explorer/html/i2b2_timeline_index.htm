Introduction

The timeline view is comprised of two tabs; (1) Create model for Timeline and (2) Render of a Timeline.  This section describes the layout and contents of both tabs.

 

Layout of the Create model for Timeline tab

The concepts to be used when rendering the timeline are defined in the Create model for Timeline tab.  Users can add individual or groups of concepts.  In addition to adding concepts, users can define value constraints for each concept and set the height and color of the ticks that appear on the timeline.

 

Figure 8. 1     Create model for Timeline Layout

 

 

Query Name

The query name will appear when you drag a previous query from either Previous Query View or Workplace View and drop it into the Query Name field.

 

Concept List

This section contains a user defined list of concepts and additional information used when creating the timeline.  The following table describes each column:

 

Column Name

Description / Use

Row #

Identifies order in which the concept will display in the timeline.

Name of Terms

The name of the concept.

Value

List of fields in the database that can be used to limit the data available on the timeline for a particular concept.

Value Text

Limit the data to a user defined range of values for a particular concept.

Height

The height of the tick on the timeline.

Color

The color of the tick on the timeline.

 

 

Deletion Buttons

Remove concepts from the list.

 

Move Buttons

Change the order of concepts on the list.

 

Display Options

Select what to display when viewing the timeline. Currently there are two options.

 

Display Concepts with No Data

A check in the box next to Display concepts with no data means all concepts listed will display in the timeline even if an event for that concept has not occurred.

 

Display Patient Demographics

A check means the patient’s demographic information will display.

No check means only the patient id will display.

 

 

Patient Set

In the bottom left corner of the timeline view is the Patient Set box.  This box will display the number of patient’s in the set that will be used when rendering the timeline.

 

 

Layout of the Render a Timeline Tab

The render a timeline tab will use the information entered on the Create model for Timeline tab.  If data has not been entered, a warning will appear and the user needs to go back and add the required information.

 

 

Figure 8. 2     Render a Timeline Layout

 

 

Timeline

Graphical display of chronological events based on the concepts identified on the Create model for Timeline tab.

 

Zoom In/Out Arrows

Arrows that allow users to zoom in and out of the timeline.

 

Increment Settings

 

Start

Defines what record number the timeline should begin the display.

Increment

Defines how many to advance when the Next and Previous buttons are clicked on.

 

Previous and Next Buttons

Next button moves the timeline forward to the next group of patients.

Previous button moves the timeline backwards to the previous group of patients.

 

 

 

Creating a Timeline

This section describes the process of adding items to the query tool, setting constraints and setting the display options for the timeline.

 

Draggable Items

In order to create your timeline, you will need to define what information will be retrieved from the databases.  Selected items from other i2b2 Workbench views can be dragged and dropped (copied) into the Timeline view.

 

1.       Concepts

Concepts () can be defined as terms, results, providers, etc.  They can be added as a group (folder/subfolder ()) or individually by dragging the item(s) from Navigate Terms, Find Terms, and Workplace views.

 

2.       Previous Query

The concepts and patient set can easily be added by dragging a previous query () from Previous Query and Workplace views.

 

3.       Group Template

Users can define a group of concepts that they frequently run and save them in their workplace view. This grouping of concepts is referred to as a group template (). This template can be added to the timeline view by dragging it from Workplace view to the concept list section.

 

4.       Patient Set

A specific set of patients can be added to the timeline by dragging the patient list () from Previous Query and Workplace views.

 

 

Add Concepts

Concepts can be added to the timeline view by simply dragging the concept from one of the following views.

Delete/Remove

There are two options for removing a concept from the list of concepts to be used. Users can (1) delete the entire listing of concepts, or (2) delete a single concept from the list.

 

Delete All Concepts

1.       Click on the Delete All button located under the list of concepts.

2.       The list of concepts will be cleared.

 

Warning: Once you click on the Delete All button, all the concepts on the list and their constraints will be removed, therefore you need to make sure you want to delete everyone.

 

Delete Single Concept

1.       Click on the concept in the list that you want to remove.

2.       Once the concept is highlighted, click on the Delete From List button.

3.       The concept will be removed from the list.

 

 

Move Concepts

Each item listed is assigned a Row # that is used when rendering the timeline. This row # determines the order the concepts will appear in the timeline. Users have the option of moving concepts to a new row before the timeline is created.  The system will automatically update the row #.

 

Move Concept Up

1.       Highlight the concept to be moved.

2.       Click on the Move Up button.

3.       Continue to click on the move up button until the record is in the desired location.

 

Move Concept Down

1.       Highlight the concept to be moved.

2.       Click on the Move Down button.

3.       Continue to click on the move down button until the record is in the desired location.

 

Put Concepts in Order

1.       Double click on the row # field for the concept to be moved.

2.       Enter the new row #.

3.       Click on the Put In Order button.

4.       The row #’s for the remaining concepts will be updated and placed in the correct sequential order.

 

 

Setting Constraints

The only constraints that can be defined in the timeline view are value constraints.  Not all concepts have values. For instance, diagnosis of Acute Myocardial Infarction does not have a value but the tests used to make the diagnosis do have values (high, low, numeric, etc.).  For those concepts that do have values there are different types of values that may be associated with it.

 

Type of Value

Description/Use

NVAL_NUM

Numerical Value

TVAL_CHAR

Character/Text Value

Location_CD

Location code

Confidence_Num

 

Location_Path

Address/path for where the data is stored

InOut_CD

Inpatient or outpatient code

 

The type of value constraint to be used is defined in the Value column.

 

Values in the timeline view use operators, such as greater than or less than when setting constraints. The following is a listing of operators currently in use.

 

Operator

Description/Use

Equal To (=)

An exact match to the value entered.

Greater Than (>)

Any value greater than the number entered.

Less Than (<)

Any value less than the number entered.

Between And

Any value in the range entered.

Like

Any value resembling the one entered.

 

The operator along with the constraint criteria is defined in the Value Text column.

 

 

SETTING THE VALUE CONSTRAINT

1.       Double click on the field in the value column.

2.       The field will change to a drop down list.

 

Figure 8. 17  Value Column

 

 

3.       Select the appropriate value.

4.       Double click on the field in the Value Text column. 

5.       The field will change to a drop down list containing the operators.

 

Figure 8. 18  Value Text Column

 

 

6.       Select the appropriate operator.

7.       Add the criteria (text) to the operator.

 

 

Display Options

Users can change how data is represented on the timeline. The things that can be changed are the height and color of the tick, whether or not display patient demographics or to include concepts with no data.

 

Height

The height of the tick can be changed for each concept and is done by changing the value in the height column of the concept list section.  The options available are:

 

§         Very Tall

§         Tall

§         Medium

§         Low

§         Very Low

 

Changing the height of a concept may be done to create a way to easily identify the concept on the timeline or to reflect the highs and lows of lab values.

 

 

Change Height

1.       Double click on the field in the Height column of the concept list.

2.       The field will change to a drop down list.

3.       Click on the down arrow to display the list of options.

 

Figure 8. 19  Height Options

 

 

4.       Select the height you want to use.

5.       The new height will appear.

 

Figure 8. 20  Height and Color

 

 

Color

The color of the tick can be changed for each concept and is done by changing the value in the color column of the concept list section.  This may be done to create a way to easily identify the concept on the timeline.

 

Change Colors

1.       Double click on the field in the Color column of the concept list.

2.       A new window will open displaying a color palette (Figure 8.21)

 

Figure 8. 21  Color Palette

 

 

3.       Click on the color you want to use.

4.       Click on the OK button.

5.       The color palette window will close and the new color will be displayed (see Figure 8.20)

 

 

Concepts without data

A check in the box next to Display concepts with no data means all concepts listed will display in the timeline even if an event for that concept has not occurred.

 

Patient Demographics

Users may choose whether or not they want the patient’s demographic information to display on the timeline.  The patient id will always display.

 

A check mark in the box next to Display Patient Demographics means the demographic information will appear in the timeline.

 

Figure 8. 22  Patient Demographics Displayed

 

 

No check mark in the box next to Display Patient Demographics means the demographic information will not appear in the timeline.

 

Figure 8. 23  No Patient Demographics Displayed

 

 

 

Viewing a Timeline

 

Navigation

 

 

Collapse/Expand Headings

Users can choose to expand or collapse a heading in the timeline by clicking on the arrow to the left of the heading.

 

Zoom In/Out

The timeline view allows the user to zoom in and out of the timeline by dragging the arrows located at the bottom of the timeline.

 

Zoom In

Users can zoom in to any point on the timeline.

 

Zoom in Towards the Beginning of the Timeline

1.       Click on the arrows icon () to the right of the timeline.

2.       Holding the Left Mouse Button down, drag the arrow to the left.

3.       Release the Left Mouse Button once you reach the point on the timeline you want to be the end point.

 

Figure 8. 24  Zoom to Beginning

 

 

4.       The dates at the top of the timeline will be updated to reflect the new range.

 

Figure 8. 25  Date Change: Zoom to Beginning

 

 

Zoom in Towards the End of the Timeline

1.       Click on the arrows icon () to the left of the timeline.

2.       Holding the Left Mouse Button down, drag the arrow to the right.

3.       Release the Left Mouse Button once you reach the point on the timeline you want to be the beginning point.

 

Figure 8. 26  Zoom to End

 

 

4.       The dates at the top of the timeline will be updated to reflect the new range.

 

Figure 8. 27  Date Change: Zoom to End

 

 

Zoom in to the Middle of the Timeline

1.       Click on the arrows icon () to the left of the timeline.

2.       Holding the Left Mouse Button down, drag the arrow to the right.

3.       Release the Left Mouse Button once you reach the point on the timeline you want to be the end point.

4.       Click on the arrows icon to the right of the timeline.

5.       Holding the Left Mouse Button down, drag the arrow to the left.

6.       Release the Left Mouse Button once you reach the point on the timeline you want to be the beginning point.

 

Figure 8. 28  Zoom to Middle

 

 

7.       The dates at the top of the timeline will be updated to reflect the new range.

 

Figure 8. 29  Date Change: Zoom to Middle

 

 

Zoom Out

Users can zoom out to any point on the timeline.

 

Zoom Out Towards the Beginning of the Timeline

2.       Holding the Left Mouse Button down, drag the arrow to the right.

3.       Release the Left Mouse Button once you reach the point on the timeline you want to be the beginning point.

4.       The dates at the top of the timeline will be updated to reflect the new range.

 

Zoom Out Towards the End of the Timeline

3.       Release the Left Mouse Button once you reach the point on the timeline you want to be the end point.

4.       The dates at the top of the timeline will be updated to reflect the new range.

 

 

Moving to Next/Previous Groups of Patients

The timeline view will display the patient set in groups. The number of patients in a group is defined by the user.

 

Increment Settings

Start

Defines what record number the timeline should begin the display.

Increment

Defines how many to advance when the Next and Previous buttons are clicked on.

 

Move Forward and Backwards

At the bottom of the Timeline View are two buttons for moving forward and backward through the patient set.

The Next button () moves the timeline forward to the next group of patients.

The Previous button () moves the timeline back to the last group of patients.

 

Notes Viewer

Notes in the timeline can be viewed by double clicking on the tick. Depending on how the system is setup the notes may be encrypted in which the user will need a key to view the note.

 

View a Note

1.       In the timeline, identify the note to be viewed.

2.       Double click on the tick.

3.       A new window will open displaying the text of the note.

 

Figure 8. 30  Notes Viewer

 

 

View an Encrypted Note (Enter Key Manually)

1.       In the timeline, identify the note to be viewed.

2.       Double click on the tick.

3.       A new window will open informing you that you need an encryption key.

 

Figure 8. 31  Encryption Key Request

 

 

4.       Click on the down arrow to display the choices in the drop down list.

5.       Select Type in the Key.

6.       Click on the OK button.

7.       A new dialog box will open asking you to type in the decryption key.

 

Figure 8. 32  Input Decryption Key

 

 

8.       Enter the key.

 

Warning: It is important to make sure you enter the key correctly. If you do not enter the key correctly a message will appear and you will need to create the timeline again in order to have another chance to enter the correct key.

 

Figure 8. 33  Invalid Key Message

 

 

9.       Click on the OK button.

10.   The notes viewer window will open displaying the note (see Figure 8.30)

 

 

View an Encrypted Note (Enter Key from a File)

1.       In the timeline, identify the note to be viewed.

2.       Double click on the tick.

3.       A new window will open informing you that you need an encryption key (see Figure 8.30).

4.       Click on the down arrow to display the choices in the drop down list.

5.       Select Browse to find the file containing the key.

6.       Click on the OK button.

7.       A new dialog box will open allowing you to browse for the key.

 

Figure 8. 34  Browse for File

 

 

8.       Find the file containing the key.

9.       Click on the OK button.

10.   The notes viewer window will open displaying the note (see Figure 8.30)

 

Close a Note

1.       Click on the exit button () in the top right corner of the notes viewer.

2.       The window will close.
